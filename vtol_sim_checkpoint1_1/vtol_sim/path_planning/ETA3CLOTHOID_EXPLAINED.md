# η³-Clothoid Planner v3.2 — 알고리즘 원리 및 수학적 분석

---

## 1. 클로소이드(Clothoid)란 무엇인가

**클로소이드(Clothoid)**, 또는 **오일러 나선(Euler Spiral)** 이라 불리는 이 곡선은
곡률(curvature)이 호의 길이(arc length)에 따라 **선형적으로 변화**하는 곡선이다.

```
κ(s) = κᵢ + (κⱼ − κᵢ) / L · s
```

여기서:

- `κᵢ`, `κⱼ` : 세그먼트 시작·끝점에서의 곡률 (1/m)
- `L` : 세그먼트 호 길이 (m)
- `s` : 호 길이 파라미터 (0 ≤ s ≤ L)

이 성질 때문에 클로소이드는 **자동차·항공기·철도 선로 설계**에서 직선에서 원호로 자연스럽게
전환하는 완화 곡선(transition curve)으로 수십 년간 사용되어 왔다.

### 왜 곡률 선형성이 중요한가?

항공기가 경로를 따를 때 조향 입력(선회율, 뱅크 각 등)은 곡률에 비례한다.
곡률이 갑자기 점프(불연속)하면 조향 입력이 급변 → **기체 응력, 탑승자 불편, 제어 불안정**.
클로소이드는 이 전환을 매끄럽게 만들어 준다.

## 2. 기본 수학: 헤딩과 위치 적분

### 2.1 헤딩(heading) 방정식

곡률을 적분하면 헤딩이 나온다:

```
θ(s) = θᵢ + ∫₀ˢ κ(t) dt
      = θᵢ + κᵢ·s + ½·(κⱼ − κᵢ)/L · s²
```

헤딩은 s의 **2차 다항식** — 부드럽고 예측 가능하다.

### 2.2 위치 방정식 (프레넬 적분)

헤딩을 다시 적분하면 위치가 나온다:

```
x(s) = ∫₀ˢ cos(θ(t)) dt
y(s) = ∫₀ˢ sin(θ(t)) dt
```

`θ(t)`가 s의 2차식이므로 이 적분은 **닫힌 형태(closed form)가 없다**.
→ 코드에서는 사다리꼴 수치 적분(trapezoidal rule)으로 처리한다.

```python
# _fresnel_endpoint() — 수치 적분으로 세그먼트 끝점 계산
s = np.linspace(0.0, L, n_quad + 1)
dk = (kappa_j - kappa_i) / L
th = theta_i + kappa_i * s + 0.5 * dk * s * s
return np.array([np.trapz(np.cos(th), s), np.trapz(np.sin(th), s)])
```

## 3. G2 연속성 — 이 알고리즘의 핵심 목표

경로가 **G2 연속** 이라는 것은:

| 연속성 등급 | 조건           | 물리적 의미               |
|-------------|----------------|---------------------------|
| G0          | 위치 연속       | 경로가 끊어지지 않음       |
| G1          | 헤딩 연속       | 갑작스러운 방향 전환 없음  |
| G2          | 곡률 연속       | 조향 입력의 급변 없음      |

G2 조건을 수식으로 쓰면, 각 웨이포인트 k에서:

```
θ_end(seg k)   = θ_start(seg k+1)    ← G1
κ_end(seg k)   = κ_start(seg k+1)    ← G2 (클로소이드에선 노드 κ값이 양쪽 공유)
pos_end(seg k) = wps[k+1]            ← 웨이포인트 정확 통과
```

모든 세그먼트에서 동시에 이 조건을 만족시키는 것이 풀어야 할 방정식 시스템이다.

## 4. 미지수(변수) 구성

N개의 웨이포인트가 있을 때, (N−1)개의 세그먼트가 생성된다.

**경계 조건 (고정):**
- 시작 헤딩 θ₀, 시작 곡률 κ₀
- 끝 헤딩 θₙ, 끝 곡률 κₙ

**최적화 변수:**
- 내부 노드 헤딩: θ₁, …, θₙ₋₁ → (N−2)개
- 내부 노드 곡률: κ₁, …, κₙ₋₁ → (N−2)개  
  (단, tanh 변환으로 |κ| ≤ κ_max 자동 보장: `κ = κ_max · tanh(u)`)
- 세그먼트 길이: L₁, …, Lₙ₋₁ → (N−1)개  
  (단, exp 변환으로 L > 0 자동 보장: `L = chord · exp(v)`)

**총 미지수 수: (N−2)·2 + (N−1) = 3N−5**

### tanh/exp 변환의 중요성

```python
# 곡률: |κ| ≤ κ_max 구조적 보장 (하드 제약 → 부드러운 변환)
kp[1:-1] = kappa_max * np.tanh(x[n_inner:2 * n_inner])

# 세그먼트 길이: L > 0 구조적 보장
Ls = np.maximum(chords * np.exp(np.clip(v, v_lo, v_hi)), 1e-3)
```

이 변환 덕분에 NR 반복 중 제약 위반이 구조적으로 불가능하다.
별도의 제약 처리 없이도 항상 물리적으로 타당한 해를 탐색한다.

## 5. 잔차 방정식 (Residual System)

각 세그먼트 k에 대해 3개의 잔차 방정식:

```
F[3k]   = x_end(seg k) − (wps[k+1][0] − wps[k][0])   ← x 위치 오차
F[3k+1] = y_end(seg k) − (wps[k+1][1] − wps[k][1])   ← y 위치 오차
F[3k+2] = w_head · wrap(θ_end(seg k) − θ_{k+1})       ← 헤딩 연속성 오차
```

여기서:
- `θ_end(seg k) = θₖ + ½(κₖ + κₖ₊₁)·Lₖ` (사다리꼴 근사)
- `w_head = min(mean_chord, 50.0)` — 위치 잔차와 헤딩 잔차의 스케일 균형
- `wrap(a) = (a + π) mod 2π − π` — 각도 연속성 보장

**총 방정식 수: 3·(N−1)**

미지수(3N−5)와 방정식(3N−3) 수의 차이 = 2 → 경계 조건 2개로 정확히 상쇄.

## 6. 뉴턴-랍슨 + 레벤버그-마르콰르트 반복법

방정식 `F(x) = 0`을 반복적으로 풀기 위해 **Levenberg-Marquardt 변형 NR**을 사용:

```
(JᵀJ + λI) · Δx = −Jᵀ F
```

여기서:
- `J` : 유한 차분(finite difference)으로 계산한 야코비안 행렬
- `λ = 1e-8 · tr(JᵀJ) / n` : 자동 조정 댐핑 계수 (특이 행렬 방지)
- 수렴 실패 시 `numpy.linalg.lstsq`로 최소제곱 폴백

### Armijo 선탐색 (line search)

단순 NR은 발산 위험이 있으므로, 각 반복에서 최적 스텝 크기를 찾는다:

```python
# Armijo 조건: 충분한 감소를 보장하는 최소 스텝
for _bt in range(20):
    if ||F(x + step·dx)|| ≤ (1 − c·step)·||F(x)||:
        break
    step *= 0.5
```

이 두 기법의 조합으로 넓은 초기값 범위에서도 안정적인 수렴을 달성한다.

### 다중 클리핑 시도 (Fallback Strategy)

```python
clip_attempts = [
    (-0.5, 1.2),   # 기본: 보수적
    (-0.7, 1.8),   # 2차: 여유 확대
    (-1.0, 2.5),   # 3차: 최대 허용
]
```

NR이 수렴하지 않으면 세그먼트 길이의 허용 범위를 단계적으로 넓혀가며 재시도.
세 번의 시도 중 잔차가 가장 작은 결과를 채택한다.

## 7. 초기값 전략

NR의 수렴 속도와 품질은 초기값에 크게 의존한다.

### 헤딩 초기값: 가중 이등분선

```python
# 두 인접 세그먼트의 방향을 길이 역제곱근으로 가중 평균
w_in  = 1 / √L_in
w_out = 1 / √L_out
bisector = w_in · (d_in/L_in) + w_out · (d_out/L_out)
θᵢ = arctan2(bisector.y, bisector.x)
```

짧은 세그먼트에 더 큰 가중치를 부여해 급격한 방향 전환에서도 좋은 초기값을 생성한다.

### 곡률 초기값: 멩거 곡률

```python
# 세 점이 이루는 삼각형의 외접원 반지름의 역수
κ = 2 · |cross(a, b)| / (|a| · |b| · |chord|)
```

이 기하학적 추정값은 국소 곡률의 자연스러운 근사치다.

## 8. 웨이포인트 자동 삽입

NR 풀기 전, 실현 불가능한 구간을 미리 감지해 중간점을 삽입한다:

```python
# 필요 곡률 추정: 호의 길이 근사
k_need = 2 · Δθ / chord
if k_need > κ_max · 0.9:
    # 웨이포인트 중앙에 새 WP 삽입
    wp_mid = 0.5 · (wps[i] + wps[i+1])
    θ_mid  = mean_angle(θᵢ, θᵢ₊₁)
```

**물리적 의미:** 두 웨이포인트 사이의 헤딩 변화가 너무 크면, 최소 선회 반경을 초과하는
곡률이 필요해진다. 이를 사전에 감지해 경로를 부드럽게 분할한다.

## 9. v3.2 핵심 혁신: Affine 보정 (선형 잔차 분배)

NR 수렴 후에도 위치 잔차가 남을 수 있다. v3.2는 이를 우아하게 처리한다:

### 문제

- 누적 좌표 방식: 세그먼트를 연결하면 잔차가 **누적**되어 후반 WP를 빗나감
- 단순 로컬 좌표: WP 통과 보장이 없음

### 해법: 선형 보정

각 세그먼트를 로컬 좌표(wps[k] 기준)로 샘플링한 후, 끝점 잔차를 호 길이에 비례해 분배:

```python
err = target_end_local - seg_pts_local[-1]   # 끝점 위치 잔차
correction = np.outer(seg_s / L_total, err)  # 선형 분배
seg_pts_corrected = seg_pts_local + correction
seg_pts_global = seg_pts_corrected + wps_2d[k]
```

**수학적 해석:**

보정량 δ(s) = (s/L) · err

- s = 0 에서: δ = 0 (시작점 고정)
- s = L 에서: δ = err (끝점이 정확히 wps[k+1]에 도달)
- 중간: 선형 보간 → 곡선 형태가 최소한으로 변형

**결과:**
- NR 잔차가 작으면 → 보정량도 작아 원래 클로소이드와 거의 동일
- NR 잔차가 커도 → 모든 WP를 정확히 통과하는 매끄러운 경로 보장

## 10. 터미널 감쇠 세그먼트

경로 끝에서 곡률이 0이 아니면, 자연스러운 직선 복귀를 위해 감쇠 세그먼트를 추가:

```python
L_decay = |κ_end| / (κ_max · 0.5)
# κ: κ_end → 0 으로 선형 감소하는 클로소이드 세그먼트
```

이후 직선 연장 세그먼트(`end_extension = 15.0 m`)로 유도 로직이 경로를 확실히 따라잡을 수 있게 한다.

```python
# 웨이포인트 고도를 호 길이(s)를 기준으로 선형 보간
alt_arr = np.interp(s_arr, wp_s_arr, wp_h_arr)
```

```python
χ(i) = arctan2(dy, dx)              # 수평 코스 각도 (course angle)
```

## 12. 코드의 우수성 분석

### 12.1 수학적 정확성

| 특성 | 방법 | 효과 |
|------|------|------|
| 곡률 연속성 (G2) | 전역 NR 시스템 | 모든 웨이포인트에서 동시 만족 |
| 최대 곡률 보장 | tanh 변환 | 구조적 하드 제약, 위반 불가 |
| 웨이포인트 통과 | Affine 보정 | NR 잔차와 무관하게 항상 보장 |
| 각도 연속성 | wrap() 함수 | π 경계에서 부호 반전 문제 없음 |

### 12.2 수치 안정성

- **Levenberg-Marquardt 댐핑**: 야코비안이 특이(singular)에 가까워도 안정적 풀이
- **lstsq 폴백**: LM이 실패해도 최소제곱 해로 우아하게 복구
- **Armijo 선탐색**: 발산 방지, 단조 감소 보장
- **다중 클리핑 시도**: 하나의 초기 설정이 실패해도 자동으로 다음 시도
- **수치 영 방지**: `max(L, 1e-3)`, `max(chord, 1e-9)` 등 분모 보호

### 12.3 물리적 타당성

- **항공기 기동 제한 준수**: `κ_max = a_max / v²` (구심가속도 한계)
- **accel_tol 여유율**: 이론 최대 곡률의 90%로 제한 → 실제 비행 안전 마진
- **자동 WP 삽입**: 실현 불가능한 기동 요구를 사전 차단

### 12.4 설계 품질

```
구조적 모듈화:
  _fresnel_endpoint()   ← 수치 적분 (재사용 가능)
  _clothoid_sample()    ← 경로 샘플링 (재사용 가능)
  _menger_kappa()       ← 초기값 추정 (재사용 가능)
  _initial_thetas()     ← 초기값 추정 (재사용 가능)
  _insert_wps_if_*()    ← 전처리 (독립적)
  _residual()           ← 잔차 계산 (테스트 가능)
  _solve_g2_nr()        ← 핵심 수치해법 (독립적)
  plan()                ← 조합 및 출력 (상위 레벨)
```

각 함수가 단일 책임을 가지며, 독립적으로 테스트하거나 교체할 수 있다.

### 12.5 성능

- 수치 적분 샘플 수를 파라미터화 (`n_quad`)하여 정확도/속도 트레이드오프 제어
- `np.trapz`, `np.outer` 등 numpy 벡터 연산 적극 활용
- 불필요한 재계산 없이 변수를 재사용

## 13. 알고리즘 전체 흐름도

```
입력: waypoints_ned (N×3), aircraft_params
         │
         ▼
[Stage 0] 전처리
  • kappa_max = a_max / v_cruise²
  • 초기 헤딩 추정: 가중 이등분선 (_initial_thetas)
  • 실현 불가능 구간 감지 → 중간 WP 자동 삽입 (_insert_wps_if_infeasible)
         │
         ▼
[Stage 1] G2 NR 풀이 (_solve_g2_nr)
  • 초기값 구성 (θ, κ via Menger, L via chord)
  • tanh/exp 변환으로 제약 내재화
  • LM-NR 반복 + Armijo 선탐색
  • 3회 클리핑 시도, 최선 선택
  • 출력: θ[0..N], κ[0..N], L[0..N-1]
         │
         ▼
[Stage 2] 경로 샘플링 + Affine 보정
  • 각 세그먼트를 로컬 좌표로 샘플링 (_clothoid_sample)
  • 끝점 잔차를 호 길이로 선형 분배 (Affine correction)
  • 글로벌 좌표 변환
         │
         ▼
[Stage 3] 터미널 처리
  • 곡률 감쇠 세그먼트 (_terminal_decay)
  • 직선 연장 세그먼트 (end_extension)
         │
         ▼
[Stage 4] 3D 합성
  • 호 길이 기반 고도 선형 보간
  • χ (코스각), γ (상승각) 계산
         │
         ▼
출력: Path (PathPoint 리스트 + 메타데이터)
```

## 15. 핵심 참고 수식 요약

```
곡률:     κ(s) = κᵢ + (κⱼ − κᵢ)/L · s
```

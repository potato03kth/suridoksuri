# η³-Clothoid Planner v3 — 구조 설명

## 0. 한눈에 보는 변경점 (v2 → v3)

| 항목 | v2 | v3 |
|---|---|---|
| 단계 수 | 2단계 (Stage1 G2 보간 + Stage2 G1 NR) | **1단계 통합 NR** |
| Stage 1 잔차 | "자연 헤딩/곡률"과의 차이 (간접) | **위치 + 헤딩 잔차 (직접)** |
| Stage 2 존재 | 별도 NR로 θ, κ, L 재조정 | **제거** (Stage 1이 모두 해결) |
| G2 강제 방식 | Picard 평활화 (근사) | **단일 클로소이드 매개변수화로 자동 보장** |
| L_seg 클립 | `(-1.0, 1.5)` → 4.48배까지 폭주 | **`(-0.3, 0.6)` → 자기 루프 원천 차단** |
| WP 삽입 시 θ | 두 WP 잇는 직선 방향 (G1 깨짐) | **양옆 θ의 원형 평균 (G1 부드러움)** |
| WP 삽입 시 κ | 0 강제 (G2 깨짐) | **삽입 후 NR이 자유롭게 결정** |
| 종단 κ 천이 | 마지막 클로소이드 → 직선 (κ 점프) | **κ 감쇠 클로소이드 1개 삽입 후 직선** |

---

## 1. 핵심 아이디어

> **모든 segment를 단일 클로소이드 (κ가 선형)로 두고, 인접 segment가 노드 κ를 공유하도록 매개변수화하면 G2는 자동으로 만족된다.**

즉 `κ_k` (노드 곡률) 자체를 자유변수로 두면:
- segment k의 끝 κ = `κ_{k+1}`
- segment k+1의 시작 κ = `κ_{k+1}`
- → **자동으로 동일** ⇒ G2 OK (별도 잔차 불필요)

남은 문제는 두 가지뿐:
1. **위치**: 각 segment 끝점이 다음 WP에 정확히 도달
2. **헤딩(G1)**: 각 segment 끝 헤딩이 다음 segment 시작 헤딩과 일치

이 둘만 잔차로 강제하면 끝.

---

## 2. 자유변수 (총 차원 `3N - 5`)

| 변수 | 차원 | 의미 | 매개변수화 |
|---|---|---|---|
| `θ_i`, i=1..N-2 | N−2 | interior 헤딩 | 직접 |
| `u_i`, i=1..N-2 | N−2 | interior 곡률 | `κ_i = κ_max · tanh(u_i)` ← **κ_max 자동 준수** |
| `v_k`, k=0..N-2 | N−1 | segment 길이 | `L_k = chord_k · exp(clip(v_k, -0.3, 0.6))` ← **자기 루프 차단** |

**경계 조건** (고정):
- `θ_0 = initial_heading`, `θ_{N-1} = (마지막 두 WP 방향)`
- `κ_0 = κ_{N-1} = 0`

---

## 3. 잔차 (총 차원 `3(N-1)`)

각 segment k에 대해 3개씩:

```
F_pos_x[k] = (∫₀^{L_k} cos θ(s) ds) − (x_{k+1} − x_k)
F_pos_y[k] = (∫₀^{L_k} sin θ(s) ds) − (y_{k+1} − y_k)
F_head[k]  = mean_chord · wrap( θ_k + ½(κ_k+κ_{k+1})·L_k − θ_{k+1} )
```

여기서 segment 내부 헤딩은 클로소이드 정의로:
```
θ(s) = θ_k + κ_k·s + ½·(κ_{k+1}-κ_k)/L_k · s²
```

**스케일 정합**: 헤딩 잔차에 `mean_chord` (m)를 곱해서 위치 잔차(m)와 단위·크기 통일 → 야코비안 조건수 개선.

## 4. 파이프라인

```
┌─────────────────────────────────────────────────────────────┐
│ 입력: waypoints_ned, aircraft_params, initial_state         │
└─────────────────────┬───────────────────────────────────────┘
                      ▼
        ┌───────────────────────────────┐
        │ 경계 조건 결정                 │
        │ • θ_0, θ_N, κ_0=κ_N=0          │
        │ • κ_max = a_max / v_cruise²    │
        └───────────────┬───────────────┘
                        ▼
        ┌───────────────────────────────┐
        │ Stage 0: WP 사전 삽입          │
        │ • 가중 이등분선으로 θ_pre 추정 │
        │ • κ_needed = 2|Δθ|/chord       │
        │   > κ_max·0.9 인 구간에 WP     │
        │   삽입 (중점, θ는 원형 평균)    │
        └───────────────┬───────────────┘
                        ▼
        ┌───────────────────────────────┐
        │ Stage 1: 진짜 G2 NR            │
        │ • 자유변수: θ_inner, u, v      │
        │ • 잔차: 위치(2) + 헤딩(1)      │
        │ • 수치 야코비안 + Armijo LS    │
        │ • G2: 매개변수화로 자동        │
        └───────────────┬───────────────┘
                        ▼
        ┌───────────────────────────────┐
        │ Clothoid 샘플링 (ds 간격)      │
        │ • segment별 좌표/κ/θ 생성      │
        └───────────────┬───────────────┘
                        ▼
        ┌───────────────────────────────┐
        │ 종단 κ 감쇠 클로소이드          │
        │ • L_decay = |κ_end|/(κ_max·0.5)│
        │ • κ_end → 0 부드러운 천이      │
        └───────────────┬───────────────┘
                        ▼
        ┌───────────────────────────────┐
        │ 직선 연장 (κ=0)                │
        └───────────────┬───────────────┘
                        ▼
        ┌───────────────────────────────┐
        │ 호 길이, 고도, χ, γ 계산        │
        │ → Path 객체 반환                │
        └───────────────────────────────┘
```

## 6. 보장 항목

| 항목 | 보장 방식 |
|---|---|
| ✅ **WP 완전 통과** | 위치 잔차 `F_pos` < `nr_tol` (기본 1e-5 m) |
| ✅ **G1 (헤딩 연속)** | 헤딩 잔차 `F_head/mean_chord` < `nr_tol/mean_chord` rad |
| ✅ **G2 (곡률 연속)** | 단일 클로소이드 + 노드 κ 공유 매개변수화 (구조적 자동) |
| ✅ **κ_max 준수** | `κ = κ_max · tanh(u)` 매개변수화로 |κ| < κ_max 영구 보장 |
| ✅ **자기 루프 없음** | `L = chord · exp(v)`, v ∈ [−0.3, 0.6] → L ∈ [0.74, 1.82]·chord |
| ✅ **종단 κ 부드러움** | 감쇠 클로소이드로 마지막 κ → 0 → 직선 |
| ❌ **G3 (κ 미분 연속)** | 미구현. 노드에서 dκ/ds 점프 허용 |

## 7. v2 증상별 해결 매핑

| v2 증상 | v3 해결 |
|---|---|
| WP1 직전/직후 끊어짐 | `V_CLIP_HI=0.6`으로 L 폭주 차단 + 위치 잔차 직접 강제 |
| WP2-3 꼬물거림 | Stage 1이 위치 잔차로 직접 풀어 κ가 의미있게 결정됨 |
| WP4 직후 경로 중복 | `V_CLIP_HI=0.6` (≈1.82배)으로 자기 루프 원천 차단 |
| WP5 진입 전 이상한 직선 | WP 삽입 시 θ는 원형 평균, κ는 NR이 결정 (직선 강제 X) |
| 직선↔곡선 연결 비-clothoid | 종단 κ 감쇠 클로소이드 추가 (κ 점프 제거) |

## 9. 알고리즘 복잡도

- **Stage 0**: O(N · max_insert) — 보통 1~5회 패스
- **Stage 1 야코비안**: O(M · n_x · n_quad) per iter
  - M = 3(N−1) 잔차, n_x = 3N−5 변수, n_quad = 100 적분점
  - per iter ≈ O(N²)
- **NR 전체**: 보통 10~20 iter → 총 O(N²) ~ O(N³)

N ≤ 50 정도까지는 실시간(<100ms)에서 잘 작동. 대규모는 sparse 야코비안 + 해석적 그래디언트로 개선 가능.

## 10. 사용 예시

```python
from eta3_clothoid_planner_v3 import Eta3ClothoidPlannerV3
import numpy as np

planner = Eta3ClothoidPlannerV3(
    ds=1.0,
    accel_tol=0.9,
    nr_tol=1e-5,
    verbose=True,
)

waypoints = np.array([
    [   0.0,    0.0,  100.0],
    [-100.0,  420.0,  120.0],
    [-600.0,  300.0,  150.0],
    [ 450.0,  400.0,  150.0],
    [  80.0, -400.0,  120.0],
    [ 700.0, -220.0,  100.0],
])

aircraft_params = {
    "v_cruise": 18.0,    # m/s
    "a_max_g":  0.3,     # g
    "gravity":  9.81,
}

initial_state = {"initial_heading": np.deg2rad(90.0)}

path = planner.plan(waypoints, aircraft_params, initial_state)

print(f"총 길이: {path.total_length:.1f} m")
print(f"점 수:   {len(path.points)}")
print(f"계획 시간: {path.planning_time*1000:.1f} ms")
```
